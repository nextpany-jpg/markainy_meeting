## 📋 사용 시나리오

### 영업사원의 하루
```
1. 아침에 시스템 접속
2. Gmail 로그인 (최초 1회)
3. 오늘 미팅 정보 입력
   - 날짜: 2026-03-10
   - 시간: 14:00
   - 장소: 서대문 경찰청
   - 고객사: 경찰청
   - 목적: 제품 소개
4. "미팅 준비 시작" 클릭
5. 자동으로 수집되는 정보 확인
   - 날씨: 서대문 지역 날씨
   - 뉴스: 경찰청 관련 최신 뉴스
   - 이메일: 경찰청 관련 이메일 요약
   - 일정: 구글 캘린더에 자동 등록
   - 맛집: 경찰청 주변 맛집 5개
6. "전체 내용 이메일로 받기" 클릭
7. 이메일 클라이언트에서 자신에게 전송
8. 미팅 전 이메일 확인하며 준비
```

## 🔧 기술 스택

### Backend
- Node.js v24.14.0
- Express.js
- Google APIs (Gmail, Calendar)
- iconv-lite (한글 인코딩)

### Frontend

- HTML5

### APIs
- Google OAuth 2.0
- Gmail API
- Calendar API
- Gemini AI API
- Kakao Maps API

## 📊 API 사용량

### Gemini API (무료 버전)
- 제한: 하루 20회
- 사용: 미팅당 약 4-5회
  - 날씨 검색: 1회
  - 뉴스 검색: 1회
  - 이메일 요약: 1회
  - 추가 웹 검색: 1-2회
- 하루 약 4-5개 미팅 준비 가능

### Gmail API
- 제한: 하루 1,000,000 할당량
- 사용: 미팅당 약 50-100 할당량
- 충분한 여유

### Calendar API
- 제한: 하루 1,000,000 할당량
- 사용: 미팅당 1-2 할당량
- 충분한 여유

### Kakao Maps API
- 제한: 하루 300,000회
- 사용: 미팅당 2-3회
- 충분한 여유

## ⚠️ 알려진 제한사항

### 1. Gemini API 할당량
- 무료 버전: 하루 20회 제한
- 초과 시: 24시간 후 리셋
- 해결: 자동 재시도 (최대 2회) 또는 새 API 키

### 2. 이메일 검색 조건
- 제목: "결과 보기(GIST 이번달 활동결과 확인용)"
- 기간: 최근 30일
- 필터: 고객사명 포함

### 3. 맛집 검색
- 반경: 1km 고정
- 개수: 5개 고정
- 정렬: 거리순

### 4. 날씨 정보
- 실시간 날씨 API 미사용
- Gemini AI의 일반 지식 활용
- 정확도 제한적
